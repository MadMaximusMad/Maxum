# Governance System

**Governance defines how system parameters evolve over time.**

Governance operates as a structured control layer through which participants can propose and approve changes to system parameters.

The governance process follows a defined sequence:

> Proposal → Voting → Timelock → Execution

Governance does not directly operate the system. It defines the constraints under which the system operates, ensuring that changes are coordinated and consistent with system design.

{% hint style="info" %}
**Interpretation**

Governance modifies system constraints, not day-to-day system operations.
{% endhint %}

The scope of those changes is defined in [Parameter Control](parameter-control.md), and their execution model is formalized in [Execution Constraints](execution-constraints.md).
