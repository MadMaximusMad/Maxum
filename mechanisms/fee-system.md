# Fee System

**The fee system converts activity into system-level capital.**

Maxum implements a transaction-layer value capture mechanism in which every interaction with deployed liquidity generates fees. This mechanism ensures that revenue is derived directly from usage rather than from emissions.

Fee distribution is structured across three coordinated outputs:

| Allocation | Function | System Effect |
| --- | --- | --- |
| Burn | Reduces circulating supply | Reinforces scarcity |
| Liquidity Distribution | Rewards liquidity providers | Sustains market depth |
| Balance Sheet Allocation | Accumulates capital | Expands deployment capacity |

Because the mechanism operates at the transaction layer, it captures value from measurable economic activity. As activity increases, fee generation increases proportionally.

{% hint style="info" %}
**Mechanism**

Value is captured at the transaction layer, not distributed through emissions.
{% endhint %}

{% hint style="success" %}
**System effect**

Higher activity directly increases capital formation.
{% endhint %}

The fee system operationalizes [Revenue Layer](../capital-flow/revenue-layer.md), rewards [Liquidity System](liquidity-system.md), and accumulates into [Balance Sheet](../capital-flow/balance-sheet.md).
