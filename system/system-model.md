# System Model

**Maxum is composed of interdependent system states that define how capital moves and evolves.**

The system can be understood through four primary components:

| Component | Definition | Role in System |
| --- | --- | --- |
| Capital | Accumulated value held within the balance sheet | Defines deployment capacity |
| Liquidity | Capital in a deployed state | Enables markets and activity |
| Markets & Activity | Environments and interactions utilizing liquidity | Generate throughput |
| Fees & Balance Sheet | Mechanism and state of capital accumulation | Reinforce system capacity |

Capital represents accumulated value within the system and exists primarily within the balance sheet. It defines the system’s capacity to deploy liquidity and is continuously formed through fees and deposits, then redeployed into markets.

Liquidity is capital in its active form. Once deployed, it enables trading, application usage, and price discovery. Liquidity is not external to the system; it is a direct expression of internal capital.

Markets are the environments in which liquidity is utilized. Activity represents all interactions occurring within those environments. Throughput generated from activity determines how effectively capital is utilized.

Fees convert activity into capital. A portion of every interaction is captured and accumulated into the balance sheet, reinforcing the system’s capital base and constraining supply through RFV.

{% hint style="info" %}
**State relationship**

Each component represents a different state or function of the same underlying economic value.
{% endhint %}
